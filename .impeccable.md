# Impeccable Design System

## Design Context

### Users
- **Retail store staff** across flexible roles — a permission system grants access to any feature based on what an admin allows
- **Primary context**: Desktop POS stations in Arabic-speaking retail environments; dense, data-heavy workflows optimized for speed
- **Job to be done**: Complete daily retail operations — selling at POS, managing inventory, purchasing from suppliers, reconciling shifts, generating reports
- **Needs**: Confidence and control over every action; errors are costly so clarity and feedback are critical

### Brand Personality
- **3 words**: Professional, Trustworthy, Modern
- **Voice**: Clean, confident, no-nonsense premium. Speaks with authority but remains approachable
- **Emotional goal**: Users should feel **in command** — the interface inspires confidence through clarity, precision, and deliberate design
- **Anti-personality**: NOT playful, colorful, casual, or generic. No flashiness for its own sake

### Aesthetic Direction
- **Visual tone**: Modern professional POS. Monochrome slate/zinc base with emerald/teal accent. Frosted glass touches, clean linework, high contrast
- **Theme**: Light-only (data-theme="global"), optimized for bright retail environments
- **Typography**: Tajawal + Noto Sans Arabic (Arabic-first), Outfit for numerals. Heavy use of 800-900 weight for hierarchy, compact sizes (11-15px base)
- **References**: No specific reference — the standard is premium retail/enterprise tools
- **Anti-references**: Avoid colorful/playful aesthetics. No generic cookie-cutter templates. No unnecessary visual noise
- **Layout**: Dense but not cluttered. Generous whitespace within structured grids. Strong typographic hierarchy guides the eye

### Design Principles
1. **Clarity Above All** — Every element must justify its existence. Labels are explicit, states are obvious, errors are visible. The user should never guess what something does.
2. **Confidence Through Feedback** — Every action triggers a visible, immediate response. Buttons depress, states change, data updates. Users feel in control because the system reacts predictably.
3. **Hierarchy-Driven Layout** — Typography weight (800-900 for titles, 600-700 for labels, 400-500 for body) and spacing create clear visual zones. The most important action on any page is visually dominant.
4. **Monochromatic + Intentional Color** — The palette is 95% slate/zinc grayscale. Emerald/teal is reserved for accent/interactive elements. Semantic colors (green, red, amber, blue) are used strictly for status signaling — never decoration.
5. **Density Without Clutter** — Compact sizing (11-15px type, 37-44px inputs) for data-rich screens, balanced with 8-24px gaps and card elevation. Every pixel of screen space earns its place.

### Accessibility
- **WCAG AA compliance** as baseline — sufficient color contrast, keyboard-navigable interfaces, visible focus rings
- **High contrast support** — text and interactive elements maintain strong contrast ratios against backgrounds
- **RTL-first** — all layouts flow right-to-left for Arabic; directional styles use `rtl:` / `ltr:` variants only
